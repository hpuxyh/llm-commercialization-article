亏的是豆包，赚的是字节？
大模型商业化真正算的是入口账

有一个很有意思的现象。

如果你同时用过不同大模型产品，可能会发现：有的产品已经可以直接帮你“点奶茶、下单、订服务”；有的产品依然更克制，更多停留在“回答问题”。

比如在一些产品里，Agent 已经能直接接入外卖、电商、支付体系，而在另一些产品里，这一步还被明显收着。

表面看，这是产品策略差异。

但如果往深一层看，这其实不是“技术路线不同”，而是商业化账本完全不同。

01 先别谈商业化，先算一笔最朴素的账

要理解大模型平台到底能不能靠“卖模型能力”活下来，需要先看一个最基础的问题：模型到底赚多少钱？成本又是多少？

我们用一个非常朴素的方式拆一下。这里不是财报，而是量级推算。

  • 日均 token 调用量：约 120 万亿 token/天
  • 换算后：约 1.2 亿个“百万 token 单位/天”
  • 每百万 token 平均收入假设：约 0.3 元

那么，每天收入约等于 1.2 亿 × 0.3 元，也就是约 3600 万元。再乘以 365 天，每年大约是 131 亿元级别。

考虑免费用户、折扣定价、不同模型混合调用，以及企业/API 的价格差异，比较合理的结论是：接口收入大致在百亿级，而不是千亿级。

真正的问题不是“百亿收入少不少”。
真正的问题是，调用量已经是超级互联网级别，但收入仍然只是百亿级。

02 更关键的问题：成本是另一条完全不同的曲线

如果只看收入，会误判。

因为大模型真正的成本结构，是随着使用深度不断放大的。一次普通问答有推理成本，长上下文会拉高成本，多轮 Agent 调用会继续叠加成本，图像和视频生成又是更重的一层。再往后，还有带宽、存储、训练和持续迭代。

  • 推理成本：每一次回答都要算力
  • 长上下文成本：读得越长，账单越厚
  • 多轮 Agent 成本：搜索、比较、调用工具、确认支付会多次消耗模型
  • 多模态成本：图像、视频、语音比普通文本更重
  • 基础设施成本：带宽、存储、训练和模型迭代持续发生

这意味着：用户用得越多，不一定利润越高，反而可能成本增长越快。

如果把模型业务做一个整体测算，2026 年收入，包括订阅、API、广告试水和早期 Agent，大约 230 亿；同期成本大约 1350 亿。结果就是：缺口约 1120 亿，覆盖率约 17%。

03 一个反直觉的结论出现了

如果只看这组数字，会得出一个直觉判断：模型业务根本不赚钱。

但更关键的地方在下一层：收入在变好，缺口反而可能变大。

原因是，收入增长和成本增长不是同一条曲线。收入是慢变量，要靠产品、付费、广告和抽佣慢慢成熟；成本是快变量，只要调用量、上下文长度、多模态和 Agent 任务上来，就会立刻放大。

所以即使未来进入成熟阶段，收入可能到 950 亿，成本也可能到 2400 亿。覆盖率从约 17% 提升到约 40%，看起来更好了；但缺口反而从 1120 亿扩大到 1450 亿。

2026 测算230 亿收入 / 1350 亿成本缺口=1350-230=1120，覆盖率=230÷1350≈17%。
成熟期测算950 亿收入 / 2400 亿成本缺口=2400-950=1450，覆盖率≈40%。

这就是最反直觉的地方：
规模越大，不代表越接近盈利，反而可能更依赖新的商业模式。

04 真正的核心问题出现了

到这里，问题可以收敛成一个关键判断：大模型平台的问题，从来不是“能不能卖 API”，而是“入口怎么赚钱”。

因为 API 只是第一阶段。真正的路径其实是三步：先卖能力，再卖曝光，最后卖结果。

  • 第一阶段：卖能力。API、订阅、企业服务。
  • 第二阶段：卖曝光。AI 答案中的推荐、品牌露出、类似搜索广告迁移。
  • 第三阶段：卖结果。Agent 直接下单、帮用户做决策、交易完成后抽佣。

05 为什么广告是必然的，而不是“想不想做”

很多人会误解一个问题：大模型为什么一定要做广告？

答案其实不是战略选择，而是结构迁移。

过去互联网是这样的：用户搜索，点击链接，比较信息，再做决策。广告挂在搜索结果、信息流、电商列表里。

但 AI 出现后，用户越来越少看十个链接，而是直接看一个答案。

这带来一个结构性变化：入口从“链接列表”变成“单一答案”。

于是广告不会消失，只会迁移：从搜索结果迁移到 AI 答案，从信息流迁移到推荐结果，从点击迁移到被“提及”。

一句话总结：
钱不会消失，只是换了入口。

06 为什么 Agent 抽佣是第二条必然路径

如果说广告是“迁移”，那 Agent 是“升级”。

关键变化是，过去用户自己做所有决策：比价、选商品、看评价、填信息、下单。未来则变成：用户只说目标，Agent 负责执行。

比如，“帮我点个附近评分最高的外卖”；“帮我订一张性价比最高的机票”；“帮我选一个适合送礼的商品”。

只要 Agent 做到三件事：更省时间、不更贵、结果可靠，用户就会接受它替代自己决策。

而一旦这件事发生，商业模式自然会从“信息收费”变成“交易抽佣”。

本质变化是：
从“帮你找信息”，变成“帮你完成结果”。

07 三个玩家，三种完全不同的路径

① 千问（阿里）

千问背后是淘宝、支付宝、本地生活和履约体系。它的优势是有完整交易闭环。

所以 Agent 对它来说，不是破坏体系，而是增强体系。它可以更激进地做“直接下单”。

② 豆包（字节）

豆包背后是内容流、广告系统、种草链路和电商转化。问题在于，Agent 会缩短“内容 → 种草 → 转化”的路径。

这意味着，用户可以直接成交，广告曝光减少，投流链路变短。豆包面临的不是单纯的产品选择，而是新入口和旧广告系统之间的内部矛盾。

这也是为什么它更克制。

③ DeepSeek

DeepSeek 没有同等级的交易系统，也没有同等级的广告体系。它更可能走低价、中立、基础模型路线。

它的优势反而是：不被商业化路径绑架。

08 真正的本质：不是模型竞争，是入口竞争

如果把所有东西压缩成一句话：大模型不是在卖能力，而是在争夺下一代入口。

入口一旦变了，搜索广告会变，内容分发会变，电商转化链路会变，甚至“决策权”本身都会被重新定价。

09 最后一个更底层的结论

回到最开始的问题：为什么看起来豆包在亏钱，但字节还在继续做？

因为你看到的是“产品账”，但它算的是“集团账”。

豆包本身可能是高成本、低直接收入、强补贴。但在集团层面，它的价值是下一代入口、广告理解能力升级、电商转化效率提升，以及长期 AI 操作系统入口。

所以“亏的是豆包，赚的是字节”，更准确地说，不是豆包亏损不重要，而是豆包不能只按一个独立 App 的短期损益来理解。它更像字节押注下一代入口的一张期权。

结语

大模型商业化的本质不是“模型怎么收费”，而是一个更大的问题：谁能定义下一代互联网入口。

当入口从搜索框变成 AI 对话框，再变成 Agent 执行链路之后，广告会迁移，交易会迁移，决策也会迁移。

最终留下来的问题只有一个：谁能在不破坏旧体系的情况下，接住新入口。
