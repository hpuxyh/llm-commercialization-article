大模型平台商业化：
从模型账本到入口账本

千问已经把“帮我点杯奶茶”做成了交易型 Agent 的样板：提出需求、比较选项、调用支付、完成履约。豆包则更克制，公开叙事仍主要围绕内容、搜索、模型能力和增值服务。这个差异不能只用“谁技术更强”解释，真正要看的，是每个平台背后的收入结构、成本结构和旧业务约束。更深一层看，互联网商业正在从“被搜索”走向“被 AI 提及”，再走向“被 Agent 选中并成交”。

先给结论：
大模型平台的商业化不是“卖会员能不能赚钱”这么简单，而是一道入口账本题：模型账本短期很难闭环，所以平台必须把收入从“卖 token”推到“卖决策入口”和“卖交易结果”。

事实锚点3.45 亿 MAUQuestMobile 披露的豆包 2026Q1 月活规模。
2026 测算230 亿收入 / 1350 亿成本覆盖率约 17%，缺口约 1120 亿。

后面的分析只围绕这 6 个数字展开：先看模型账本为什么打不平，再看钱为什么会从搜索广告迁移到 AI 答案广告和 Agent 抽佣，最后解释为什么千问更积极、豆包更克制、DeepSeek 更适合基础设施定位。事实、假设和判断会分开处理：事实来自可查来源；未披露项目用公式和基准情景测算；战略判断则来自“用户入口迁移 → 收费对象变化 → 平台旧业务约束 → 用户信任边界”这条推理链。

数据口径：事实、假设和推断要分开看

数据来源：事实锚点来自：QuestMobile 2026Q1 AI 应用报告、火山引擎豆包产品页、火山引擎公布 120 万亿 token 的公开报道、火山方舟模型价格页、DeepSeek 官方 API Pricing、科技日报关于千问接入淘宝/支付宝/淘宝闪购的报道、支付宝 AI 付官网。

计算口径：推断链条为：公开事实锚点 → 变量公式 → 平台战略判断。收入、成本、覆盖率和抽佣额不能写成公司事实；只能在读者认可假设时，用公式自行替换参数。

一、商业化不是一条路，而是四类收费对象

讨论大模型怎么赚钱，不能只问“卖会员还是卖 API”。平台真正面对的是四类付费对象：用户、开发者、品牌/商家、企业组织。收费对象不同，商业化形态、落地节奏和风险完全不同。

这也是商业化节奏的底层原因：用户订阅和 API 调用最像已有软件/云服务，产品可以先跑；广告要等 AI 答案成为稳定决策入口；Agent 抽佣则要等支付、履约和售后闭环成熟；企业云和行业解决方案虽然付费意愿更明确，但交付更重、周期更长。

向用户收费订阅、增值能力、生产力工具。优点是现金流直接，问题是国内 C 端付费率和客单价有天花板。
向开发者收费API、模型调用、上下文、工具链。优点是规模化，问题是价格战和推理成本会压薄毛利。
向品牌/商家收费AI 答案广告、推荐排序、Agent 成交分成。优点是接近互联网旧收入池，问题是会触碰用户信任和旧业务。
向企业组织收费云、私有化、行业解决方案。优点是付费意愿更明确，问题是交付周期长、项目制重。

这里还有一个容易被高估的收入池：AI 替代客服、办公、营销运营等企业人力成本。它当然有价值，但不应该被算进广告或 Agent 抽佣，而应该归入 toB 云、行业应用和解决方案。更重要的是，市场规模不等于平台收入：被替代的人力成本，要经过“AI 可替代率、企业愿意支付比例、应用层与模型/云层分成”三层折扣，最后才是模型平台能拿到的钱。

因此，合理的节奏不是“先盈利再扩张”，而是先跑通低摩擦收费，再把平台入口和交易结果变成可收费资产：订阅和 API 先证明付费，AI 答案广告承接旧预算，Agent 抽佣绑定交易结果，云和行业解决方案沉淀长期利润。

数据口径 1：商业化节奏的依据

数据来源：API 价格以火山方舟模型价格页和 DeepSeek 官方 Pricing为准；交易闭环参考科技日报关于千问接入淘宝、淘宝闪购、支付宝的报道、支付宝 AI 付官网、OpenAI Instant Checkout / ACP 官方说明和 Stripe 关于 ACP 的官方新闻稿；广告预算迁移参考36 氪相关行业趋势报道与秒针《中国数字营销趋势报告》。

计算口径：这里不是公司指引，而是产品成熟度排序：已商品化（订阅/API）→ 可承接预算（AI 答案广告）→ 需交易闭环（Agent 抽佣）→ 需企业交付（云/解决方案）。2026-2028 这类时间点只能理解为推断节奏。

二、为什么必须商业化：模型账本天然不容易闭环

大模型的财务难点不在“有没有人用”，而在“使用越深，成本越硬”。一次普通问答、一次长上下文推理、一次图片生成、一次工具调用和一次 Agent 执行，背后都是不同强度的算力、带宽、存储和工程成本。用户规模扩大以后，平台首先得到的往往不是利润，而是更大的推理账单。

这笔账也不能只理解为“训练一次大模型花多少钱”。训练投入只是起点，真正持续发生的是推理成本、模型迭代、上下文变长、多模态生成、Agent 多步调用、带宽存储、工程研发和获客运营。尤其是 Agent 场景，一次“帮我买”背后可能包含搜索、比价、读取评价、调用工具、确认支付、生成解释等多轮推理；用户感觉路径变短，平台背后的调用链反而变长。

这张图的关键不是给出一个“豆包亏多少”的确定答案，而是说明怎么计算才合理。公开数据能够证明豆包有很大的使用规模和用户规模，但收入要看多少调用真正付费、按什么加权单价计费；成本则涉及推理、训练、带宽、存储、研发和获客，公司没有逐项披露。

所以专业写法不是直接断言“豆包一年亏多少”，而是：在官方 token 量和官方价格页基础上，建立 API 收入公式；在公司未披露成本的情况下，只能做成本敏感性分析。只要任何一个关键变量变化，结论都会变化。

但文章不能只有公式，也要给读者一个量级感。沿用前面对话里的基准情景，可以这样估：2026 年如果把订阅、API、广告试水和少量企业服务放在一起看，收入约 230 亿；成本端如果把算力资本开支/折旧、训练迭代、推理服务、研发和获客运营一起算，约 1350 亿，缺口约 1120 亿。到 2027-2028 年，四条腿成熟后收入可能抬到约 950 亿，但成本也可能随 token、多模态和 Agent 调用扩张到约 2400 亿，缺口仍约 1450 亿。

这组数的读法：它不是“豆包官方亏损”，而是一个可替换参数的基准情景。真正重要的不是 1120 亿或 1450 亿这两个绝对值，而是方向：收入覆盖率可能从约 17% 升到约 40%，但如果成本随规模更快上行，绝对缺口仍会扩大。

这也是为什么“卖模型能力”不够。订阅的上限取决于用户付费意愿，API 的上限受价格战约束，而广告和交易抽佣来自更大的商业预算池。大模型平台必须从能力供应商，升级成入口、决策和交易基础设施。

数据口径 2：模型账本的事实与测算

数据来源：火山引擎豆包产品页公开标注日均 tokens 使用量，新浪科技/爱范儿报道也称火山引擎公布豆包大模型日均 Token 使用量突破 120 万亿；QuestMobile披露 2026 年 3 月豆包、千问、DeepSeek 月活分别为 3.45 亿、1.66 亿、1.27 亿；火山方舟模型价格页和 DeepSeek Pricing提供模型调用价格口径。

计算口径：API 年收入公式：日 token 量 × 365 ÷ 1,000,000 × 加权单价 × 付费化比例。以 120 万亿 token/日为锚点，全年约 438 亿个“百万 token 单位”；若等效加权单价按 0.3 元/百万 token 左右估算，可得到约 128 亿 token 收入量级。总收入约 230 亿还要加订阅、企业服务和广告试水；成本约 1350 亿按算力/折旧、训练、推理、研发、获客运营分项估。成熟期收入约 950 亿来自四条腿加总，成本约 2400 亿来自用量和 Agent 多步调用继续增长。

三、平台广告为什么会出现：钱会跟着决策入口迁移

广告预算本质上不是跟媒体形态走，而是跟用户决策入口走。搜索时代，品牌买关键词；信息流时代，品牌买推荐流量；AI 时代，如果用户直接看答案，品牌自然会关心“答案里有没有我、排序靠不靠前、推荐理由是否可信”。

用户迁移的底层逻辑，也不是“大家突然喜欢 AI”。而是每一代入口都替用户多做一步：搜索给的是链接和原料，用户还要自己读、自己比、自己判断；对话式 AI 给的是整理后的答案，用户少做了阅读和归纳；Agent 给的是完成后的结果，用户连比较、下单、支付和售后入口都可以少切换。入口每往下游走一步，商业预算也会跟着往下游走一步。

这张图是全文题眼。品牌侧有三次迁移：从被搜索，到被 AI 提及，再到被 Agent 买单。平台侧也对应三次迁移：从搜索广告，到 AI 答案广告，再到 Agent 抽佣。所谓“大模型平台广告”，不是凭空创造的新蛋糕，而是旧的搜索、内容、电商广告预算在新入口上的再分配。

站在平台方视角，“做 GEO”和“做平台广告”其实是同一枚硬币的两面。第三方 GEO 服务商想做的是自然优化：让品牌在 AI 答案里被提及、被引用、被推荐；平台自己不会长期把这套黑箱优化方法白白交出去，因为那会伤害答案可信度。平台更可能做的是把“可见性”产品化：官方品牌数据接入、认证档案、赞助答案、推荐位、品牌解释卡、可标识的商业推荐，以及未来的 Agent 成交分成。换句话说，平台不会去当传统 SEO 代运营，它会把 GEO 想争取的那部分可见性收费化。

这里要把“GEO”拆成三类，否则很容易把不同玩家的动作混在一起。入口强势的平台，不会优先做中立 GEO 服务商；它会把自己入口里的可见性收编成广告和交易抽佣。入口弱势或工具型公司，才更可能去卖“帮你在各家 AI 里被提及”的服务。还有第三类是甲方自用 GEO：像内容、电商、本地生活平台，会给自己的商品、内容、知识库做结构化改造，争取在豆包、DeepSeek、元宝、千问等外部 AI 答案里被引用。

A. 平台卖可见性拥有入口的平台把 AI 答案位、推荐位、品牌卡和成交结果商业化，本质是广告和抽佣。字节、百度、阿里、腾讯更接近这一类。
B. 第三方 GEO 服务入口不强或工具型公司帮品牌提升在多家 AI 里的被提及、被引用和被推荐概率，本质是服务商或 SaaS。
C. 甲方自用 GEO有大量内容、商品或本地生活资产的公司给自己做结构化内容、官方知识库和 AI 友好数据，守住“被 AI 引用权”。

所以，未来大厂的真实动作不一定叫“GEO”。更值得盯的是巨量引擎、百度营销、阿里妈妈、腾讯广告会不会上线 AI 答案位、AI 推荐位、品牌认证档案、Agent 成交归因这类商业化产品。名字可能是“AI 营销”“智能投放”“品牌知识库”“商品数据接入”，但底层都是同一件事：把 AI 决策入口里的可见性和成交结果变成可收费资产。

更可能的产品节奏，也不是一上来就在答案里塞满广告。第一步会是“官方数据接入”：让品牌、商家、商品、门店和服务以更可靠的格式进入平台知识库；第二步是“认证档案和赞助推荐”：在不破坏答案可信度的前提下，把商业身份、推荐理由和标识规则做清楚；第三步才是“成交归因和抽佣”：当 Agent 真能下单、预约、支付和售后时，平台按结果收费。

这也解释了为什么平台必须谨慎。AI 答案越像“唯一建议”，广告越接近决策干预；商业化能力越强，用户信任风险也越高。未来平台广告的核心竞争点，不只是能不能卖广告位，而是能不能在“可解释推荐”和“商业排序”之间建立足够稳定的边界。

数据口径 3：广告迁移与 GEO 三类玩家

数据来源：36 氪《2026 AI 重构下的社交媒体营销趋势报告》相关报道与秒针《中国数字营销趋势报告》用于证明广告主和营销服务商正在关注 AI 化入口；QuestMobile用于证明 AI 原生 App 已形成大规模用户入口；千问接入淘宝、淘宝闪购、支付宝的事实参考科技日报报道和支付宝 AI 付官网，用于说明交易闭环已经进入产品化阶段。

计算口径：计算上不直接把搜索广告盘子全部搬到 AI。更稳妥的迁移公式是：可迁移广告收入 = 旧搜索/内容/电商广告预算 × AI 入口渗透率 × 可商业化展示率 × 平台分成率。第三方 GEO 服务、甲方自用 GEO、平台广告/抽佣是相关但不同的收入池，不能互相等同。

四、Agent 抽佣为什么是胜负手：它卖的是结果，不是流量

如果说 AI 答案广告仍然是在影响注意力，那么 Agent 抽佣就是直接参与结果。搜索给用户一堆链接，AI 答案给用户一个建议，Agent 则把任务推进到支付、履约和售后。平台从“卖曝光”变成“卖完成”，商业模型也从注意力经济进入意图经济。

这里最关键的一句话是：被提及不等于被选中，被选中也不等于能成交。一个品牌可能出现在 AI 答案里，但如果商品数据不完整、价格不透明、库存不可读、优惠不可核验、支付和售后不能接入 Agent 流程，它在成交环节仍然等于不存在。Agent 时代优化的不只是“让人看到你”，而是“让机器能理解你、比较你、信任你，并替用户完成交易”。

这也是为什么 Agent 商业化会倒逼商家做“机器可读”的基础设施。人买东西会被图片、品牌情绪、直播话术影响；Agent 更看重三件事：数据是否完整，接口是否可访问，交易协议是否能闭环。规格、价格、库存、配送、退换、优惠券、评价摘要如果不能被稳定读取，就算品牌在答案里被提到，也很难进入最后的购买候选。

数据完整规格、适用场景、价格、库存、优惠和售后规则要结构化，否则 Agent 不敢替用户做确定性选择。
机器可读最好有 API、MCP 或标准化 feed，让 Agent 不靠猜页面、不靠抓取残缺信息就能读取商品和服务。
协议可成交支付、授权、发票、履约和售后要能接入 Agent 流程；否则只是被看见，不能被买单。
成交分佣Agent 帮用户完成购买、预订、投保、开户或本地服务履约后，平台按成交结果从商家侧收取分成。
商业推荐商家为“被推荐、被比较、被优先解释”付费，但必须明确标识并受信任边界约束，否则会伤害答案可信度。
履约服务费支付、身份核验、售后、发票、物流、客服等工具能力也可以收费，本质是把 Agent 交易链路做成基础设施。

Agent 抽佣的想象空间更大，是因为它挂在 GMV 上，而不是挂在 token 上。但这并不意味着平台能把全部 GMV 都算成新增收入。真正的收入要扣掉履约成本、商家补贴、支付和售后成本，更重要的是，还要扣掉从自己旧业务里“搬运”过来的那部分收入。

所以判断 Agent 抽佣，不能只看“能促成多少 GMV”，要看净贡献：这笔成交是不是新增的、原来会不会在平台旧链路里发生、旧链路本来能赚多少广告和佣金、Agent 直达后还剩多少收费空间。对有电商和广告基本盘的平台来说，Agent 可能既是新收入，也是旧收入的压缩器。

如果沿用一个偏乐观的基准情景，四条腿成熟期可以拆成约 950 亿：订阅约 150 亿、API 约 420 亿、AI 答案广告约 220 亿、Agent 抽佣约 160 亿。这组数的意义不是“预测公司财报”，而是让读者看到量级：API 仍是短期大头，广告是承接旧预算的兜底项，Agent 抽佣天花板最高但起量最慢。

基准情景怎么估：订阅按 3.45 亿 MAU、低个位数付费率和约 100 元/月上下的 ARPU 推；API 按日 token 量继续增长、加权单价接近价格战后的低价区间、付费化比例提升推；广告按 AI 入口渗透率、可商业化展示率和广告预算迁移比例推；Agent 抽佣按可促成 GMV、约 8% 左右可收费率和非自我蚕食比例推。任何一个参数换掉，结果都会变。

因此，四条腿可以给出量级，但必须和计算口径绑在一起。订阅看 MAU、付费率和 ARPU；API 看 token 量、加权单价和付费化比例；广告看 AI 入口流量与商业化展示率；Agent 看可促成 GMV、take rate 和非自我蚕食比例。它们不是同一种收入，不能用一个统一倍数粗暴估算。

数据口径 4：四条变现腿的测算

数据来源：订阅的用户规模锚点来自 QuestMobile；API 单价来自火山方舟模型价格页和 DeepSeek 官方 Pricing；Agent 履约和支付闭环参考科技日报报道的千问接入淘宝/支付宝/淘宝闪购、支付宝 AI 付官网、OpenAI Instant Checkout / ACP 官方说明和 Stripe 官方新闻稿。

计算口径：订阅收入 = MAU × 付费率 × ARPU，示例约 3.45 亿 × 3%-4% × 1000-1200 元/年，得到百亿级；API 收入 = token 量 × 加权单价 × 付费化比例，示例按用量增长和付费化提升得到约 420 亿；广告收入 = AI 入口流量 × 商业化展示率 × CPC/CPM，示例按旧广告预算小比例迁移得到约 220 亿；Agent 净收入 = 可促成 GMV × take rate × 非自我蚕食比例 - 履约/补贴/售后成本，示例按 2000 亿可促成 GMV × 8% 左右可收费率得到约 160 亿。

五、豆包为什么要做：产品账本和集团账本不是一回事

如果只看豆包自己的独立账本，它可能很难好看。但平台公司做超级入口时，通常不会只看单个 App 的短期损益，而会看集团账本：它能不能守住入口，能不能提升广告和电商效率，能不能带动云和行业客户，能不能保留下一代入口的期权。

所以“亏的是豆包，赚的是字节”更准确的意思是：豆包自己的模型账本可能承压，但它对字节的价值不只体现在豆包 App 里。它可能体现在抖音搜索和内容推荐的防御、广告系统的理解能力、电商转化效率、火山引擎的模型服务，以及多模态生产力工具的长期期权里。

不要把“万亿市场”直接写成平台收入。
以 AI 客服、AI 销售、AI 办公这类效率替代场景为例，更合理的计算链条是：平台可捕获收入 = 被替代的人力/软件支出 × AI 可替代率 × 企业愿意支付比例 × 模型/云层分成。应用层、集成商、云厂商和模型平台会一起分这块钱，模型平台通常只能拿其中一层。

举一个纯假设的数字例子（非任何公司数据）：某块人力替代市场名义上有 1 万亿，但 AI 当前只能可靠替代其中一部分，企业只愿意为其中一部分付费，模型平台又只能分到应用、集成、云之外的一层——三道折扣连乘之后，真正能被平台记进账的可能只剩百亿量级，而且大多记在 toB 云，而不是广告或 Agent 抽佣。这正是“市场规模”和“平台收入”必须分开看的原因：换一组替代率和分成假设，结论就会变，所以本文只给折扣链路，不给某个固定金额。

这不是说任何亏损都合理。真正的判断标准是：如果不做 AI 入口，集团旧业务被迁移走的风险有多大；如果做了 AI 入口，它能带来多少防御、增效和新收入。豆包值不值得做，不是单看“豆包能不能盈利”，而是看它能不能降低集团未来被截流的概率。

数据口径 5：两本账的边界

数据来源：豆包用户规模参考 QuestMobile；token 调用量参考火山引擎豆包产品页与火山引擎公布 120 万亿 token 的公开报道；模型价格参考火山方舟模型价格页。字节/抖音广告收入、电商 GMV、资本开支等不是字节官方披露，本文不把它们写成事实数据。

计算口径：集团账本的计算不是“豆包亏损 + 字节收入”相加，而是风险收益判断：AI 入口防御价值 = 被截流风险 × 旧业务利润池；增效价值 = 广告/电商基数 × AI 带来的效率提升；云价值 = 企业客户用量 × 云/模型服务毛利。三个变量都需要公司数据才能精确测算。

六、为什么千问更积极，豆包更克制：自我蚕食税

同样是 Agent 商业化，千问会更积极，豆包会更克制，DeepSeek 会走另一条路。差异不主要来自技术，而来自旧业务。新入口如果吃掉自己的旧收入，公司就会慢；如果新入口是在放大旧收入，公司就会快。

这里可以用一个变量模型理解“自我蚕食税”：同样一笔消费，如果它先经过内容种草、广告投流、直播间或货架转化，平台可能同时获得广告、佣金和推荐排序收益；如果 Agent 直接替用户比价下单，用户体验更短，但原来的曝光、比较、投流环节会被压缩，平台可商业化的环节也会变少。

自我蚕食税的公式：
传统链路收入 = GMV × 传统综合货币化率 α；Agent 直达收入 = GMV × Agent 可收费率 β。若 β 小于 α，差额 GMV × (α - β) 就是平台推进 Agent 时要承担的自我蚕食税。α 和 β 必须来自具体业务数据，不能凭空写成行业平均。

把 α、β 代成一个纯假设的场景会更直观（非任何公司数据）：一笔 ¥1000 的订单，如果走传统“逛”的链路，平台可能拿到约 15% 的投流广告加约 5% 的基础佣金，合计约 20%；如果 Agent 替用户直接比价下单，省掉了“逛”，原来约 15% 的投流大部分消失，平台只能靠约 5% 的基础佣金，再加上为“被优先推荐”新设的约 3% 竞价加权，合计约 8%。中间约 12 个百分点的差额，就是平台推进 Agent 时要自己消化的自我蚕食税。需要强调的是，这里的 8% 不是硬上限——它取决于竞价加权能补回多少、新增成交有多少是纯增量；旧业务里广告和投流越赚钱的平台，这道税越重，也就越要算清楚新增成交能不能补回被压缩的旧收入。

数据口径 6：自我蚕食税公式

数据来源：可验证的事实锚点是：千问已经接入淘宝、淘宝闪购和支付宝 AI 付，证明交易型 Agent 的路径已经出现；支付宝 AI 付官网说明智能体支付、支付 MCP 等能力正在被产品化。字节内容电商和信息流广告的具体货币化率并非官方披露，本文不使用固定百分比。

计算口径：自我蚕食税 = GMV × (传统综合货币化率 α - Agent 可收费率 β) × 被替代比例。只有 α、β 和被替代比例来自具体业务数据时，才可以落到金额；否则只能作为机制公式。

千问：交易闭环优先阿里本来就有淘宝、支付宝、闪购、出行和本地生活履约体系。Agent 下单是把入口前移，不是推翻旧业务，所以更容易积极推进。
豆包：内容消费入口优先字节最值钱的是内容流里的种草、广告和转化。Agent 直达成交会缩短链路，也可能削薄投流和曝光收入，因此更需要控制节奏。
DeepSeek：基础设施优先DeepSeek 更适合占据低价、中立、开放模型和 API 心智。它不一定亲自做交易抽佣，但可能成为大量垂直 Agent 背后的模型能力。

如果豆包真的把 AI 答案广告和 Agent 抽佣推起来，千问大概率不会被动防守，而会继续把淘宝、支付宝、闪购、本地生活和出行的交易闭环前置到对话入口里。DeepSeek 则更可能走相反路线：强化低价、中立、开放模型和 API 基础设施心智，把“少商业干扰”变成差异化。也就是说，豆包做得越商业化，市场上越会出现“更会成交的 Agent”和“更干净的模型基础设施”两类反向定位。

未来竞争格局大概率不是一家通吃，而是三层并存：平台型 Agent 掌握用户入口和交易闭环，垂直 Agent 深耕具体行业和任务，模型基础设施提供低成本、高性能、中立的底层能力。最后胜负不只由模型排行榜决定，而由入口、履约、信任、成本和旧业务约束共同决定。

七、用户为什么会接受 Agent：不是为佣金付费，而是为确定性付费

用户不会因为“平台可以抽佣”而选择 Agent。用户选择 Agent，是因为它把搜索、比较、下单、支付、售后这些分散动作压缩成一个结果。只要结果足够好，人就很难退回更麻烦的路径。

省时间不用搜、看、比、切 App，任务从十几分钟压缩到一句话。
省决策用户不想研究所有选项，只想要一个足够可靠的选择。
省操作比价、优惠券、地址、支付、售后都能自动串起来。
可追责授权、支付、售后、解释机制做清楚，用户才会把 Agent 当成服务，而不只是工具。

所以“付佣金”从用户体验上看，并不是用户主动掏钱给 Agent，而是商家为成交付费，平台为用户提供更短路径。用户真正比较的是总体验：价格是否合理、推荐是否可信、售后是否可靠、有没有更省事。

用户体感大概率会经历三个阶段：第一阶段是“真香”，因为免费、方便、少切 App；第二阶段是“起疑”，因为用户会问“它给我选的，是最适合我的，还是佣金最高的”；第三阶段是“分层”，一部分用户继续接受免费层里的广告和抽佣，一部分用户愿意为更少干扰付费，还有一部分会转向低商业化、中立或开源的产品。

这也意味着 Agent 商业化的边界一定是信任。一旦用户觉得推荐被广告和佣金污染，就会转向付费去广告、开源中立、或者更垂直的专业 Agent。未来很可能形成三层产品：免费层带广告/抽佣，付费层买更高信任，中立层用低商业化建立差异。

判断口径 7：用户接受 Agent 的条件

数据来源：可参考千问接入淘宝闪购与支付宝 AI 付的公开案例：用户一句话发起需求，系统完成推荐、比价、优惠券、确认支付等流程；这证明交易型 Agent 的用户体验路径已经被样板化。

计算口径：用户是否接受 = 节省时间价值 + 决策质量提升 + 操作成本下降 + 售后可追责 - 价格溢价 - 广告干扰 - 信任损耗。这里不是财务公式，而是用户体验判断框架。

结尾：真正的竞争不是模型强弱，而是谁能重写入口账本

回到开头的问题：为什么大模型平台必须做广告和 Agent？因为单纯的模型账本很难支撑超级入口级别的成本；而广告和 Agent 抽佣，分别对应“影响决策”和“完成交易”两个更大的收入池。

  • 商业化形式：订阅、API、AI 答案广告、Agent 抽佣、toB 云和行业解决方案，本质上对应四类收费对象。
  • 节奏判断：订阅/API 先跑，广告承接旧预算，Agent 抽佣在支付和履约成熟后放量，云和解决方案沉淀长期利润。
  • 平台为什么必须做：模型账本收入薄、成本硬，必须从卖能力升级为卖入口、卖决策、卖结果。
  • 豆包的关键：不只是豆包 App 能不能赚钱，而是字节能不能守住 AI 时代的新入口和旧商业化链路。
  • 竞争格局：千问抢交易闭环，豆包抢内容消费入口，DeepSeek 抢低价中立基础设施和开放模型心智，垂直 Agent 抢具体场景。
  • 用户为什么会迁移：不是因为喜欢佣金，而是因为 Agent 用更短路径、更少决策和更强确定性换取用户信任。
  • 终局判断：免费层、付费高信任层和中立基础设施层会长期并存；商业化越强，信任治理越重要。
最终，AI 商业化拼的不是谁更会聊天，而是谁能在不摧毁用户信任的前提下，把“帮用户做决定”变成新的商业入口。
